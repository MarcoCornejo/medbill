## Core Principle

Medical documents are the most sensitive data a person has. The architecture makes data retention IMPOSSIBLE, not just unlikely.

## Rules

1. **Never store document content.** No filesystem writes, no database inserts, no cache entries containing extracted text, images, or field values.

2. **Never log PII.** Logs may contain: document type, page count, processing time, error counts, confidence scores. Logs must NEVER contain: patient names, dates of birth, addresses, SSNs, member IDs, provider names, dollar amounts, medical codes from a specific document.

3. **Never transmit document data.** The core processing pipeline makes zero network calls. Any future network feature (analytics, updates) must only transmit anonymous aggregates.

4. **Only synthetic data in tests.** All test fixtures use data generated by MedBillGen. Never copy real medical documents into the repo, even "anonymized" ones.

5. **Only anonymous counters in the database.** The SQLite database stores: date, documents_scanned (int), errors_flagged (int), estimated_savings_cents (bigint), appeals_generated (int). Nothing else. No user IDs, no session IDs, no IP addresses.

6. **Purge after processing.** After results are returned to the user, all in-memory references to the document image, extracted text, and structured data must be explicitly deleted.

7. **No cookies, no tracking, no fingerprinting.** The web interface uses zero client-side tracking. No Google Analytics, no Mixpanel, no session cookies. Optional: Plausible/Umami (self-hosted, cookieless, GDPR-compliant).

8. **Disclaimer in UI.** Every interface (web, CLI) must display: "This tool is not legal or medical advice. Consult a professional for your specific situation."

9. **Upload security.** Uploads must: enforce a size limit (currently 10MB), sanitize filenames with `Path(name).name`, close the UploadFile immediately after reading, and process content via `io.BytesIO` (never write to disk). See `web-app.md` for the full upload flow.
