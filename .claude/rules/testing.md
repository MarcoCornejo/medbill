## Test Conventions

- All test files: `tests/test_{module}.py` mirroring `src/billshield/{module}.py`
- Use `pytest` with `-v -x` (verbose, stop on first failure)
- Pin random seeds to `42` for reproducibility
- No network calls in unit tests — mock external services
- Integration tests marked with `@pytest.mark.integration`

## Data Rules

- ONLY use synthetic data generated by MedBillGen
- NEVER use real patient data, even "anonymized" samples
- Test fixtures go in `tests/fixtures/` (generated, not hand-crafted)
- Commit small fixtures (<100KB); generate large ones via MedBillGen

## Assertions

- Assert exact values, not ranges (deterministic pipeline + pinned seeds = exact results)
- For dollar amounts: compare to the cent (`assert amount == Decimal("3247.80")`)
- For extracted codes: exact string match, case-sensitive
- For confidence scores: use `pytest.approx()` with tolerance

## Quality Gates

- All new code must have tests
- Coverage target: 80% minimum
- `make lint && make test` must pass before any commit
