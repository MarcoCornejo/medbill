# MedBillBench — Medical Billing Document Understanding Benchmark

**Version:** 1.0.0
**License:** CC-BY-4.0
**HuggingFace:** `billshield/medbillbench` *(planned)*

## Motivation

No public benchmark exists for medical billing document understanding. Existing benchmarks (OmniDocBench, DocVQA, FUNSD) evaluate general documents. None address:

- Domain-specific codes (CPT, HCPCS, ICD-10, CARC, RARC)
- Financial precision (dollar amounts to the cent)
- Temporal criticality (appeal deadlines)
- Layout diversity (every hospital/insurer uses a different format)
- Error detection (extraction must be accurate enough to find errors in the source)

## Dataset

500 synthetic documents generated by MedBillGen, with perfect ground-truth annotations.

| Type | Count | Templates | Difficulty |
|---|---|---|---|
| Medical bill | 200 | 12 layouts | Clean 40%, Moderate 35%, Hard 25% |
| EOB | 200 | 7 formats | Clean 40%, Moderate 35%, Hard 25% |
| Denial letter | 100 | 6 categories | Clean 40%, Moderate 35%, Hard 25% |

## MedBillScore (0-100)

```
MedBillScore = (
    0.10 * Classification Accuracy +
    0.25 * Code Extraction F1 +
    0.25 * Amount Accuracy +
    0.15 * Date Extraction F1 +
    0.10 * Name Extraction F1 +
    0.10 * Structural Accuracy +
    0.05 * Denial Field F1
) * 100
```

## Evaluation Rules

1. No training on test set
2. Deterministic evaluation (pinned seeds)
3. Report hardware and software versions
4. API-based models allowed (report API version and date)
5. Post-processing allowed (document what you do)

## Baselines

| Model | Type | Size | Expected Range |
|---|---|---|---|
| BillShield-OCR (ours) | Fine-tuned GLM-OCR | 0.9B + LoRA | 87-92 |
| GLM-OCR (base) | Open-source VLM | 0.9B | 78-84 |
| GPT-4V | Commercial VLM | Unknown | 76-82 |
| AWS Textract | Commercial OCR | Unknown | 68-74 |
| Google Document AI | Commercial OCR | Unknown | 70-76 |
| Tesseract + regex | Traditional OCR | N/A | 40-50 |

## Limitations

- Synthetic documents only (V1). Real-document validation planned for V2.
- U.S. billing formats only.
- English only.
- 25 templates may not capture full real-world formatting diversity.
- Does not evaluate handwritten annotations.
